# 多项目升级方案

> 当有第二个推广项目需要接入Bot时，按此文档操作。

## 目标架构

```
统一入口 → 选择项目 → 对应项目知识库 + 需求 + 数据
                        ├── AI任务中台
                        ├── 项目B
                        └── 项目C ...
```

## 改动清单

### 1. 数据层：加 project_id

所有数据文件按项目隔离：

```
data/
├── projects.json              # 项目注册表
├── project_ai_task/           # AI任务中台
│   ├── requests.json          # 需求
│   ├── chat_logs.json         # 问答日志
│   ├── visits.json            # 访问记录
│   └── docs_index.json        # 文档索引
├── project_b/                 # 项目B
│   ├── requests.json
│   ├── chat_logs.json
│   ├── visits.json
│   └── docs_index.json
knowledge_base/
├── ai_task.json               # AI任务中台知识库
├── project_b.json             # 项目B知识库
```

### 2. 后端改动（app.py）

```python
# 新增：项目管理
PROJECTS_FILE = os.path.join(DATA_DIR, "projects.json")

    """每个项目独立数据目录"""
    return os.path.join(DATA_DIR, f"project_{project_id}")

# 所有数据读写函数加 project_id 参数：

```

### 3. API 改动：所有接口加 project_id

```
POST /api/chat                    → body 加 project_id
GET  /api/quick_questions         → ?project_id=xxx
POST /api/feedback                → body 加 project_id
GET  /api/requests                → ?project_id=xxx
GET  /api/kb                      → ?project_id=xxx
POST /api/docs/upload             → form 加 project_id
GET  /api/analytics/overview      → ?project_id=xxx
```

新增项目管理API：
```
GET    /api/projects              # 项目列表
POST   /api/projects              # 创建项目
PUT    /api/projects/<id>         # 编辑项目
DELETE /api/projects/<id>         # 删除项目
```

顶部加项目切换下拉框：

- 创建后自动生成空知识库和数据目录
- 上传该项目的FAQ和文档

## 升级步骤（研发执行）

1. **创建 projects.json**，把现有 AI任务中台 作为第一个项目导入
2. **迁移现有数据**：把 data/ 下的文件移到 data/project_ai_task/ 下
3. **迁移知识库**：knowledge_base.json 移到 knowledge_base/ai_task.json
4. **改 app.py**：所有函数加 project_id 参数（约改20处）
5. **改前端**：使用者端加项目选择页，管理后台加项目切换
6. **测试**：确认 AI任务中台 数据不丢失，新项目能创建

## 预估工作量

| 模块 | 工时 |
|------|------|
| 后端改动 | 4-6小时 |
| 前端改动 | 3-4小时 |
| 数据迁移 | 1小时 |
| 测试 | 1-2小时 |
| **合计** | **约1-1.5天** |

## 新项目接入流程（升级后）

运营人员在管理后台操作，无需研发：

1. 管理后台 → 项目管理 → 创建新项目
2. 上传该项目的FAQ（或手动添加）
3. 上传该项目的文档资料
4. 配置该项目的话术（项目经理名字等）
5. 把链接发给机构使用者

整个过程 **10分钟内完成**。
