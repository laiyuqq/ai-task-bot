# AI任务中台推广支持Bot — 部署文档

## 项目简介

面向客服回访场景的推广支持Bot，包含：知识库问答（支持DeepSeek大模型）、需求收集、文档上传、访问统计、管理后台。

- 使用者入口：聊天问答 + 提需求
- 管理后台：数据看板、需求管理、知识库管理、文档上传、问答记录

## 部署步骤

### 1. 安装依赖

```bash
pip install -r requirements.txt
```

### 2. 配置环境变量

| 变量名 | 必填 | 说明 |
|--------|------|------|
| `DEEPSEEK_API_KEY` | 否 | DeepSeek API密钥，配置后启用大模型问答；不配则用关键词匹配 |
| `DEEPSEEK_MODEL` | 否 | 模型名，默认 `deepseek-chat` |
| `DATA_DIR` | 否 | 数据存储目录，默认 `./data`；生产环境建议指定可写目录 |
| `PORT` | 否 | 监听端口，默认 `5000` |

### 3. 本地启动（测试）

```bash
python app.py
# 访问 http://localhost:5000/        使用者端
# 访问 http://localhost:5000/admin    管理后台
```

### 4. 生产部署（Gunicorn）

```bash
gunicorn app:app --bind 0.0.0.0:5000 --workers 2 --timeout 120
```

### 5. Nginx反向代理（接入公司域名）

```nginx
# 在公司现有网站的Nginx配置中添加一个location
location /bot/ {
    proxy_pass http://127.0.0.1:5000/;
    proxy_set_header Host $host;
    proxy_set_header X-Real-IP $remote_addr;
    proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
    proxy_set_header X-Forwarded-Proto $scheme;
}
```

配置后访问 `https://公司域名/bot/` 即为使用者入口，`https://公司域名/bot/admin` 为管理后台。

如果用独立子域名（如 `bot.company.com`）：

```nginx
server {
    listen 80;
    server_name bot.company.com;

    location / {
        proxy_pass http://127.0.0.1:5000;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
    }
}
```

### 6. 进程守护（systemd）

```bash
# /etc/systemd/system/ai-task-bot.service
[Unit]
Description=AI Task Bot
After=network.target

[Service]
User=www-data
WorkingDirectory=/path/to/ai-task-bot
Environment="DEEPSEEK_API_KEY=你的key"
Environment="DATA_DIR=/path/to/ai-task-bot/data"
ExecStart=/path/to/gunicorn app:app --bind 0.0.0.0:5000 --workers 2 --timeout 120
Restart=always

[Install]
WantedBy=multi-user.target
```

```bash
sudo systemctl enable ai-task-bot
sudo systemctl start ai-task-bot
```

## 功能说明

### 使用者端（/）
- 聊天问答：支持自然语言提问，DeepSeek大模型基于知识库回答，支持多轮对话
- 快捷问题：一键点击常见问题
- 提需求：填写机构、联系人、类型、描述，提交后项目经理跟进

### 管理后台（/admin）
| Tab | 功能 |
|-----|------|
| 数据看板 | 今日/总访问量、问答数，7天趋势图 |
| 需求管理 | 需求列表、筛选、改状态、导出Excel |
| 知识库 | FAQ增删改查，即时生效 |
| 文档资料 | 上传txt/md/pdf/docx文档，自动提取文本作为问答知识来源 |
| 问答记录 | 每条问答日志：时间、IP、内容、来源、耗时 |

### API列表

| 方法 | 路径 | 说明 |
|------|------|------|
| POST | /api/chat | 问答接口（支持history多轮对话） |
| GET | /api/quick_questions | 快捷问题列表 |
| POST | /api/feedback | 提交需求反馈 |
| GET | /api/requests | 需求列表（支持筛选） |
| POST | /api/requests/<id>/status | 更新需求状态 |
| DELETE | /api/requests/<id> | 删除需求 |
| GET | /api/stats | 需求统计 |
| GET | /api/export_excel | 导出Excel |
| GET/POST/PUT/DELETE | /api/kb | 知识库FAQ管理 |
| GET/POST/DELETE | /api/docs | 文档上传管理 |
| GET | /api/analytics/overview | 访问统计总览 |
| GET/DELETE | /api/analytics/chat_logs | 问答日志 |

## 注意事项

1. **数据存储**：当前用JSON文件存储，生产环境如需更高可靠性建议迁移到数据库（SQLite/PostgreSQL）
2. **DeepSeek配置**：不配置API Key也能运行（降级为关键词匹配），配置后体验更好
3. **知识库修改即时生效**：管理后台改了FAQ，不需要重启服务
4. **文档上传**：上传的文档存在 `DATA_DIR/docs/` 目录，文本提取后作为大模型上下文

## 后续扩展指南

### 无需改代码（运营人员在管理后台操作）

| 要加什么 | 怎么操作 |
|---------|---------|
| 加新问答 | 管理后台 → 知识库 → 新增问答 |
| 改已有问答 | 管理后台 → 知识库 → 编辑 |
| 上传文档当知识 | 管理后台 → 文档资料 → 上传文档 |
| 查看访问数据 | 管理后台 → 数据看板 |
| 查看问答记录 | 管理后台 → 问答记录 |
| 跟进/导出需求 | 管理后台 → 需求管理 |

### 需要研发改代码（按优先级）

#### 1. 接数据库（建议优先）
- **原因**：JSON文件存储在数据量大后会慢，且服务重启可能丢失
- **改动**：`app.py` 中 `_load_json` / `_save_json` 替换为数据库读写
- **建议**：SQLite（轻量）或 PostgreSQL（正式）

#### 2. 加自动通知
- **原因**：有新需求时自动通知项目经理，不用时时盯后台
- **改动**：`api_feedback` 函数中加 webhook 调用（企业微信/钉钉/邮件）

#### 3. 加用户登录/权限
- **原因**：各机构只看自己的数据，管理后台需要权限控制
- **改动**：加 Flask-Login，管理后台加登录页，可对接公司SSO

#### 4. 接企业微信/钉钉
- **原因**：使用者不用开网页，直接在IM里聊
- **改动**：加 webhook 路由，接收IM消息转给 `/api/chat`，回复推回IM

#### 5. 导出问答记录
- **原因**：按周收口时需要导出问答记录Excel
- **改动**：参照 `/api/export_excel` 加一个 `/api/export_chat_logs`

#### 6. 更换大模型
- **原因**：换其他模型或升级
- **改动**：改环境变量 `DEEPSEEK_MODEL`，或改 `call_deepseek` 函数中的 API 地址和参数
