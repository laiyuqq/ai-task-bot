# AI任务中台推广支持Bot — 部署交接文档

> 本文档供研发同事参考，照此操作即可完成部署。

---

## 一、项目概述

| 项 | 说明 |
|----|------|
| **用途** | 公司AI项目推广支持Bot，替项目经理响应机构使用者的咨询和需求 |
| **代码仓库** | `https://github.com/laiyuqq/ai-task-bot` |
| **技术栈** | Python 3.8+ / Flask / DeepSeek API |
| **部署方式** | Gunicorn + Nginx 反向代理 |

**核心功能：**
- 使用者端：聊天问答（DeepSeek大模型）+ 提需求反馈
- 管理后台：数据看板、需求管理、知识库管理、文档上传、问答记录、数据分析

## 二、快速部署（4步）

### 第1步：拉代码

```bash
git clone https://github.com/laiyuqq/ai-task-bot.git
cd ai-task-bot
```

### 第2步：装依赖

```bash
# 建议用虚拟环境
python3 -m venv venv
source venv/bin/activate

pip install -r requirements.txt
```

依赖清单（`requirements.txt`）：

| 包 | 用途 |
|----|------|
| flask | Web框架 |
| gunicorn | 生产服务器 |
| openpyxl | Excel导出 |
| requests | 调用DeepSeek API |
| pdfplumber | PDF文档解析 |
| python-docx | Word文档解析 |

### 第3步：配环境变量

| 变量名 | 必填 | 值 | 说明 |
|--------|------|-----|------|
| `DEEPSEEK_API_KEY` | ✅ | （项目经理单独提供） | DeepSeek API密钥，配了才有大模型问答 |
| `DEEPSEEK_MODEL` | ❌ | `deepseek-chat`（默认） | 模型名 |
| `DATA_DIR` | ❌ | `/opt/ai-task-bot/data`（建议） | 数据存储目录，需可写（JSON模式用） |
| `PORT` | ❌ | `5000`（默认） | 监听端口 |
| `DB_HOST` | ✅ | `192.168.5.157` | PostgreSQL数据库地址 |
| `DB_PORT` | ❌ | `15432`（默认5432） | PostgreSQL端口 |
| `DB_NAME` | ✅ | `ai_center` | 数据库名称 |
| `DB_USER` | ✅ | `langzi` | 数据库账号 |
| `DB_PASS` | ✅ | （项目经理单独提供） | 数据库密码 |

> ⚠️ `DEEPSEEK_API_KEY` 必须配置，否则Bot降级为关键词匹配，体验差很多。
> ⚠️ `DB_HOST`/`DB_NAME`/`DB_USER`/`DB_PASS` 配置后数据存PostgreSQL；不配则降级为JSON文件存储。应用启动时会自动建表，首次运行会从 knowledge_base.json 导入初始知识库。

### 第4步：启动

```bash
# 测试启动
python app.py

# 生产启动
gunicorn app:app --bind 0.0.0.0:5000 --workers 2 --timeout 120
```

启动成功后：
- 使用者端：`http://服务器IP:5000/`
- 管理后台：`http://服务器IP:5000/admin`

## 三、接入公司域名（Nginx）

### 方案A：放现有网站路径下

在公司现有 Nginx 配置中加一个 location：

```nginx
location /bot/ {
    proxy_pass http://127.0.0.1:5000/;
    proxy_set_header Host $host;
    proxy_set_header X-Real-IP $remote_addr;
    proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
    proxy_set_header X-Forwarded-Proto $scheme;
}
```

访问地址：
- 使用者端：`https://公司域名/bot/`
- 管理后台：`https://公司域名/bot/admin`

### 方案B：独立子域名

```nginx
server {
    listen 80;
    server_name bot.公司域名.com;

    location / {
        proxy_pass http://127.0.0.1:5000;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
    }
}
```

访问地址：
- 使用者端：`https://bot.公司域名.com/`
- 管理后台：`https://bot.公司域名.com/admin`

> 建议 HTTPS：加 Let's Encrypt 证书或公司现有证书。

## 四、进程守护（systemd）

让服务开机自启、挂了自动重启：

```ini
# /etc/systemd/system/ai-task-bot.service
[Unit]
Description=AI Task Bot
After=network.target

[Service]
User=www-data
WorkingDirectory=/opt/ai-task-bot
Environment="DEEPSEEK_API_KEY=（项目经理单独提供）"
Environment="DB_HOST=192.168.5.157"
Environment="DB_PORT=15432"
Environment="DB_NAME=ai_center"
Environment="DB_USER=langzi"
Environment="DB_PASS=（项目经理单独提供）"
Environment="DATA_DIR=/opt/ai-task-bot/data"
ExecStart=/opt/ai-task-bot/venv/bin/gunicorn app:app --bind 0.0.0.0:5000 --workers 2 --timeout 120
Restart=always
RestartSec=5

[Install]
WantedBy=multi-user.target
```

```bash
sudo systemctl daemon-reload
sudo systemctl enable ai-task-bot
sudo systemctl start ai-task-bot
# 查看状态
sudo systemctl status ai-task-bot
# 查看日志
sudo journalctl -u ai-task-bot -f
```

## 六、API 接口一览

| 方法 | 路径 | 说明 |
|------|------|------|
| POST | `/api/chat` | 问答（支持多轮对话，传 history） |
| GET | `/api/quick_questions` | 快捷问题列表 |
| POST | `/api/feedback` | 提交需求反馈 |
| GET | `/api/requests` | 需求列表（支持筛选） |
| POST | `/api/requests/<id>/status` | 更新需求状态 |
| DELETE | `/api/requests/<id>` | 删除需求 |
| GET | `/api/stats` | 需求统计 |
| GET | `/api/export_excel` | 导出需求Excel |
| GET/POST/PUT/DELETE | `/api/kb` | 知识库FAQ管理 |
| GET/POST/DELETE | `/api/docs` | 文档上传管理 |
| GET | `/api/analytics/overview` | 访问统计总览 |
| GET/DELETE | `/api/analytics/chat_logs` | 问答日志 |
| GET | `/api/analytics/insights` | 数据分析（机构排行/热门问题/知识盲区/响应时间/活跃时段） |

## 七、注意事项

| # | 事项 | 说明 |
|---|------|------|
| 1 | **DeepSeek Key 必须配** | 不配则降级为关键词匹配，体验差很多 |
| 2 | **DATA_DIR 要可写** | 需求、日志、文档都存在这个目录，确保有写权限 |
| 3 | **知识库改了不用重启** | 管理后台改FAQ/传文档即时生效，无需重启服务 |
| 4 | **数据存储** | 配了DB环境变量用PostgreSQL，没配降级为JSON文件。启动时自动建表 |
| 5 | **PDF解析依赖** | `pdfplumber` 需要 `gcc` 和 `libjpeg-dev`，如安装失败：`apt install gcc libjpeg-dev` |
| 6 | **超时设置** | DeepSeek API 偶尔较慢，gunicorn timeout 已设 120 秒 |
